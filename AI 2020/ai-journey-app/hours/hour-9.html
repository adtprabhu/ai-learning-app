Part 1 — Concepts (0–15 min)

Working with documents

Students have textbooks. Make your app read them. Learn RAG (Retrieval-Augmented Generation) — the technique behind every AI that answers questions from your own documents.

📚

RAG explained

Retrieval-Augmented Generation: (1) Split document into chunks. (2) When user asks a question, find the most relevant chunks. (3) Feed those chunks to Claude. (4) Claude answers from the chunks, not from training.

✂️

Chunking

Breaking a PDF into ~500 word overlapping pieces. Overlap (e.g. 100 words shared between chunks) prevents answers from being cut off at chunk boundaries.

🗺️

Embeddings (simplified)

A way to convert text into a list of numbers that represents its 'meaning'. Similar meanings = similar numbers. Used to find which chunks are relevant to a question. For simplicity, we start with keyword matching before adding embeddings.

🔧

PDF extraction

Use pdf.js (browser) or pdfplumber (Python) to extract text from PDFs. The extracted text is then chunked and indexed.

javascript
// Simple RAG implementation — keyword-based retrieval
// (Good enough for Hour 9; semantic search added in advanced hour)

function chunkText(text, chunkSize = 500, overlap = 100) {
  const words = text.split(' ');
  const chunks = [];
  let i = 0;
  while (i < words.length) {
    const chunk = words.slice(i, i + chunkSize).join(' ');
    chunks.push({ text: chunk, startWord: i });
    i += chunkSize - overlap; // overlap for continuity
  }
  return chunks;
}

function findRelevantChunks(chunks, query, topK = 3) {
  const queryWords = query.toLowerCase().split(' ')
    .filter(w => w.length > 3); // ignore tiny words

  return chunks
    .map(chunk => ({
      ...chunk,
      score: queryWords.filter(w => chunk.text.toLowerCase().includes(w)).length
    }))
    .sort((a, b) => b.score - a.score)
    .slice(0, topK);
}

async function answerFromDocument(chunks, question) {
  const relevant = findRelevantChunks(chunks, question);

  if (relevant[0].score === 0) {
    return "I couldn't find information about that in the uploaded document. Try rephrasing or check a different chapter.";
  }

  const context = relevant.map((c, i) => `[Excerpt ${i+1}]\n${c.text}`).join('\n\n');

  const prompt = `You are a study tutor. Answer the student's question using ONLY the textbook excerpts below.
If the answer isn't in the excerpts, say so.

TEXTBOOK EXCERPTS:
${context}

STUDENT QUESTION: ${question}

Answer clearly and concisely. Reference which excerpt you used.`;

  const response = await fetch("https://api.anthropic.com/v1/messages", {
    method: "POST",
    headers: { "Content-Type": "application/json", "x-api-key": process.env.CLAUDE_API_KEY, "anthropic-version": "2023-06-01" },
    body: JSON.stringify({ model: "claude-sonnet-4-20250514", max_tokens: 600, messages: [{ role: "user", content: prompt }] })
  });

  const data = await response.json();
  return data.content[0].text;
}
Exercise 1Extract text from a PDF

Use the pdfjsLib browser library (cdnjs.com has it) to extract text from a user-uploaded PDF. Log the full extracted text. How readable is it? Does formatting like tables survive?

pdf.js extracts raw text — tables and complex formatting become plain text, often messy. This is normal. For production, you'd clean the text (remove extra whitespace, fix encoding). For now, raw extraction works well enough for continuous prose textbooks.
Exercise 2Test your RAG

Upload a chapter of any textbook PDF. Ask 3 questions — one whose answer is clearly in the text, one that's borderline, one that's definitely not there. Does the system correctly answer, partially answer, and decline?

Good RAG should: confidently answer on-topic questions, say 'I found partial information' for borderline ones, and clearly say 'this isn't in the document' for off-topic ones. Keyword matching works well for exact terms but misses paraphrased concepts — that's where embeddings (more advanced) would help.
Part 3 — Review (45–60 min)

Check your understanding

Answer all questions to see your score.

Q1. What does the 'overlap' in text chunking prevent?
✅ Without overlap, an answer that falls on the boundary between chunk 1 and chunk 2 would be split — neither chunk contains the full answer. Overlap (sharing 100 words between consecutive chunks) solves this.
Q2. A student asks about a topic not covered in their uploaded chapter. Your RAG system should:
✅ Transparency is crucial in educational apps. If the answer isn't in the document, say so — don't hallucinate. You can then offer to search Google (your Hour 8 integration) as a follow-up.
Q3. Why is RAG better than just sending the entire textbook to Claude?
✅ Two reasons: (1) even large context windows have limits — a full textbook may exceed them; (2) cost — you pay per token, so sending 3 relevant excerpts is far cheaper than sending 300 pages. RAG also keeps Claude focused on the relevant content.
