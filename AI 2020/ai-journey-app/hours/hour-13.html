Part 1 — Concepts (0–15 min)

Google Translate + AI

One API call removes the language barrier. Learn to integrate Google Translate so your Study Helper works for students anywhere in the world, in their own language.

🌐

Google Translate API

Supports 100+ languages. Accepts text, returns translation. Can also auto-detect the source language. Free tier: 500,000 characters/month.

🎯

Strategy: translate at the edges

Don't translate everything. Translate user input TO English (Claude works best in English), then translate Claude's output TO the user's language. Keep the AI layer in English.

🔍

Language detection

Google Translate can detect what language the user is writing in. Use this to auto-switch the whole app — no language dropdown needed.

✅

What to translate

Translate: user questions, Claude answers, quiz questions, feedback messages. Don't translate: code snippets, proper nouns, technical terms the user typed themselves.

Part 2 — Hands-on Practice (15–45 min)

Build it

javascript
// Translate text using Google Translate API
async function translate(text, targetLang, sourceLang = 'auto') {
  const response = await fetch(
    `https://translation.googleapis.com/language/translate/v2?key=${GOOGLE_TRANSLATE_KEY}`,
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        q: text,
        target: targetLang,
        source: sourceLang === 'auto' ? undefined : sourceLang,
        format: 'text'
      })
    }
  );
  const data = await response.json();
  return data.data.translations[0].translatedText;
}

// Detect language of input
async function detectLanguage(text) {
  const response = await fetch(
    `https://translation.googleapis.com/language/translate/v2/detect?key=${GOOGLE_TRANSLATE_KEY}`,
    {
      method: 'POST',
      body: JSON.stringify({ q: text })
    }
  );
  const data = await response.json();
  return data.data.detections[0][0].language; // e.g. 'fr', 'es', 'zh'
}

// Full multilingual study session
async function multilingualStudy(userQuestion) {
  // 1. Detect what language the student is using
  const userLang = await detectLanguage(userQuestion);

  // 2. Translate question to English for Claude
  const englishQuestion = userLang !== 'en'
    ? await translate(userQuestion, 'en', userLang)
    : userQuestion;

  // 3. Ask Claude in English (always better quality)
  const englishAnswer = await askClaude(englishQuestion);

  // 4. Translate answer back to user's language
  const localAnswer = userLang !== 'en'
    ? await translate(englishAnswer, userLang, 'en')
    : englishAnswer;

  return { answer: localAnswer, detectedLang: userLang };
}

Q1. Why should you translate user input TO English before sending to Claude rather than prompting Claude in the user's language?
✅ Claude does understand many languages, but its training was dominated by English text. For nuanced educational explanations, translating to English and back gives better, more accurate results than prompting directly in another language — especially for less common languages.
Q2. Google Translate free tier is 500,000 characters/month. Each study session translates roughly 3,000 characters. How many free sessions per month?
✅ 500,000 / 3,000 ≈ 166 sessions/month on the free tier. After that, it's $20 per million characters — roughly $0.06 per session. Very affordable. For most small apps, the free tier lasts a long time.
Q3. A French student types a question. Your app detects 'fr' and translates to English. Claude answers in English. You should now:
✅ Translate back to the user's language. The whole point of the integration is that students can study in their own language. Always complete the loop: their language → English → Claude → English → their language.
