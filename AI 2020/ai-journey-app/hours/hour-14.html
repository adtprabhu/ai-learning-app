Part 1 — Concepts (0–15 min)

Tracking progress with AI

Data without insight is just noise. Learn to visualise student progress and use Claude to transform raw quiz scores into personalised, actionable study recommendations.

📈

What to track

Score per session, topics covered, which questions were wrong, time between sessions, improvement over time. Each metric tells a different story.

🧮

Use code for numbers

Never ask Claude to calculate percentages, averages, or scores. Do that in JavaScript. Only ask Claude to interpret the numbers — 'student improved 20% in physics, what does this suggest?'

🎨

Simple charts

Chart.js (free, CDN) makes bar and line charts with 10 lines of code. No D3 expertise needed. A score-over-time line chart is the most useful for students.

🧠

AI personalisation

Feed Claude a student's history summary. Ask it to identify patterns and suggest next steps. This is the 'AI coach' feature — the most motivating part of the app.

Part 2 — Hands-on Practice (15–45 min)

Build it

javascript
// Build progress data from Firebase history
function buildProgressSummary(sessions) {
  const byTopic = {};

  sessions.forEach(s => {
    if (!byTopic[s.topic]) byTopic[s.topic] = { attempts: 0, totalPct: 0, scores: [] };
    byTopic[s.topic].attempts++;
    byTopic[s.topic].totalPct += s.percentage;
    byTopic[s.topic].scores.push(s.percentage);
  });

  return Object.entries(byTopic).map(([topic, data]) => ({
    topic,
    attempts: data.attempts,
    average: Math.round(data.totalPct / data.attempts),
    trend: data.scores.length > 1
      ? data.scores[data.scores.length-1] - data.scores[0]  // positive = improving
      : 0
  }));
}

// Render a bar chart using Chart.js
function renderProgressChart(summary) {
  const canvas = document.getElementById('progress-chart');
  const ctx = canvas.getContext('2d');

  new Chart(ctx, {
    type: 'bar',
    data: {
      labels: summary.map(s => s.topic),
      datasets: [{
        label: 'Average Score (%)',
        data: summary.map(s => s.average),
        backgroundColor: summary.map(s =>
          s.average >= 80 ? '#4ade80' : s.average >= 60 ? '#fbbf24' : '#f87171'
        )
      }]
    },
    options: {
      responsive: true,
      scales: { y: { min: 0, max: 100 } },
      plugins: { legend: { display: false } }
    }
  });
}

// AI coaching: pattern analysis
async function getCoachingInsight(summary) {
  const summaryText = summary.map(s =>
    `${s.topic}: avg ${s.average}%, ${s.attempts} attempt(s), trend: ${s.trend > 0 ? '+' : ''}${s.trend}%`
  ).join('\n');

  const prompt = `You are an encouraging study coach. Here is a student's performance summary:
${summaryText}

Give personalised coaching in 3 sentences:
1. Celebrate their strongest area.
2. Gently identify what needs work.
3. Give one specific, actionable suggestion for their next study session.
Keep it warm, encouraging, and specific.`;

  const response = await fetch("https://api.anthropic.com/v1/messages", {
    method: "POST",
    headers: { "Content-Type": "application/json", "x-api-key": process.env.CLAUDE_API_KEY, "anthropic-version": "2023-06-01" },
    body: JSON.stringify({ model: "claude-sonnet-4-20250514", max_tokens: 200, messages: [{ role: "user", content: prompt }] })
  });
  const data = await response.json();
  return data.content[0].text;
}
Exercise 1Build the progress dashboard

Create a page that loads the student's Firebase history, builds the summary using buildProgressSummary(), renders the bar chart, and shows the AI coaching message. All three should appear on one clean page.

Layout: greeting at top ('Welcome back, Alex!'), coaching message in a coloured card, bar chart below it, then a table of recent sessions. The chart bars should be green (≥80%), yellow (60-79%), red (<60%) based on score — this instant visual feedback is very motivating.
Exercise 2Test the coaching quality

Log 5 fake study sessions for different topics with varying scores into Firebase. Run getCoachingInsight(). Is the advice specific and helpful? What happens if all scores are high? What if all are low?

High scores: Claude should celebrate and suggest going deeper or exploring related topics. Low scores: Claude should be encouraging, not discouraging — 'You're building a foundation in X, focusing on Y will help everything click.' If the coaching feels generic, add more specific data to the summary (e.g. which specific subtopics were wrong).
Part 3 — Review (45–60 min)

Check your understanding

Answer all questions to see your score.

Q1. Why should you calculate scores and percentages in JavaScript rather than asking Claude?
✅ LLMs are pattern-matchers, not calculators. They can make arithmetic mistakes, especially with larger numbers. Always use JavaScript (or any code) for calculations. Give Claude the already-calculated numbers and ask it to interpret them.
Q2. A student's average score in Physics went from 45% to 72% over 3 sessions. Your coaching prompt should:
✅ Specific positive feedback is far more motivating than generic praise. Your summary should include the trend (+27%), so Claude can reference it specifically. 'You've made huge progress in Physics — from 45% to 72% in just 3 sessions' is far more impactful than 'good job.'
Q3. Chart.js bar colours are green/yellow/red based on score. What design principle does this apply?
✅ Traffic light colours (green=good, yellow=warning, red=attention needed) are universally understood. Students instantly know which topics need work without reading numbers. Simple colour conventions dramatically improve usability.
