Part 1 — Concepts (0–15 min)
\n

Prompts are just detailed instructions

\n

A prompt is simply the text you send to an AI. But the way you write it makes an enormous difference — like the difference between telling a new employee "do the report" versus giving them a full brief with examples, format requirements, and context.

\n
Real-world analogy

Imagine hiring a brilliant freelancer who is new to your company. They\'re smart enough to do almost anything — but only if you explain what you need clearly. Vague brief = mediocre output. Detailed brief = excellent output. Prompting is exactly this.

\n

The anatomy of a great prompt

\n
👤

Role / Persona

Tell Claude who it should act as. \'You are a patient maths teacher who explains concepts step by step.\' This shapes tone, vocabulary, and approach.

🎯

Task

What exactly do you want? Be specific. \'Write 5 multiple-choice quiz questions about photosynthesis\' beats \'write a quiz\'.

📋

Context

Background information the AI needs. \'The student is 14, has just finished their first biology lesson, and struggles with technical terms.\'

📄

Format

How should the output look? \'Return as a JSON array. Each item has: question, options (array of 4), correct_index (0-3), explanation.\'

🎨

Examples

Show don\'t tell. Giving one example of what you want dramatically improves results. This is called \'few-shot prompting\'.

\n

System prompts vs user prompts

\n

When you build an app (not just chatting), you\'ll use two types of prompts:

\n
🔧

System prompt

The background instruction set — like the employee handbook. Defines who Claude is, what it can and can\'t do, how it should respond. Set once, applies to the whole conversation.

💬

User prompt

What the user actually types. This is the dynamic part that changes every time. Claude sees both the system prompt and user prompt together.

📝

Example

System: \'You are a friendly quiz tutor. Only discuss science topics. Always ask a follow-up question.\' User: \'Tell me about black holes.\' — Claude will respond in the tutor persona, stay on science, and ask a follow-up.

\n
⚠️

The single biggest prompting mistake: being too vague. \'Help me study\' tells Claude almost nothing. \'Create 5 quiz questions on chapter 3 of biology — mitosis — aimed at a 14-year-old, in multiple-choice format with explanations\' gets you something useful.

Part 2 — Hands-on Practice (15–45 min)
\n

The prompt upgrade drill

\n

We\'ll take one weak prompt and progressively improve it, seeing the difference each upgrade makes.

\n
1

Start weak

In Claude, type: \'Help me study biology.\' Read the response. It\'s probably very generic.

2

Add a role

Try: \'You are a patient biology tutor. Help me study biology.\' Better, but still vague.

3

Add specificity

Try: \'You are a patient biology tutor. Create 5 quiz questions about mitosis for a 14-year-old student.\' Much better.

4

Add format

Try: \'You are a patient biology tutor. Create 5 multiple-choice quiz questions about mitosis for a 14-year-old. For each question include: the question, 4 options labelled A-D, the correct letter, and a one-sentence explanation.\' This is now a production-quality prompt.

5

Add an example

Prepend: \'Here is an example of the format I want: [paste a sample question in your desired format].\' With an example, Claude almost perfectly matches your intended structure.

\n
\n
Exercise 1Write your first system prompt
\n
\n

Write a system prompt for the Smart Study Helper app we\'ll build in this course. It should define: (1) who Claude is in this app, (2) what topics it covers, (3) how it should respond when confused, (4) the tone it should use.

\n \n \n
Example: \'You are StudyBot, a friendly and encouraging AI tutor for students aged 12–18. You help students understand and remember any school subject. When you don\'t know something or a question is outside your expertise, say so honestly and suggest the student ask their teacher. Always explain concepts in simple language first, then offer a more detailed explanation if the student wants one. End every explanation with a short quiz question to check understanding.\'
\n
\n
\n
\n
Exercise 2Few-shot magic
\n
\n

Ask Claude to generate a quiz question about \'the water cycle\' WITHOUT giving an example. Then ask again, but first show it one example question in your preferred format. Compare the two outputs — how much more consistent is the second one?

\n \n \n
The second version should closely mirror your example\'s structure, vocabulary level, and style. This is few-shot prompting — you\'re showing the model exactly what \'good\' looks like. In production apps, you\'ll almost always use at least one example.

Part 3 — Review (45–60 min)
\n

Key takeaways from Hour 2

\n
✅

Be specific

Role + Task + Context + Format + Examples = a great prompt.

✅

System prompts set the stage

They define Claude\'s persona and rules for your whole app.

✅

Examples beat instructions

Showing one example output is more powerful than describing it in words.

✅

Iterate quickly

Prompting is a conversation. If the first attempt is off, refine and retry.

\n

Check your understanding

Answer all questions to see your score.

Q1. What is the main purpose of a system prompt in an AI app?
✅ Correct! The system prompt is the persistent instruction set — like an employee handbook. It shapes every response the AI gives, while the user prompt changes with each message.
Q2. You ask Claude to \'write something for my app\' and the output is mediocre. What is most likely the fix?
✅ Correct! Vague prompts produce generic outputs. Adding a role (who Claude is), format (how the output should look), context (background info), and an example dramatically improves quality.
Q3. What does \'few-shot prompting\' mean?
✅ Correct! Few-shot means giving the AI a few examples of exactly what you want. It\'s one of the most powerful prompting techniques — the model patterns-matches to your examples.
