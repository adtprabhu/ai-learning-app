Part 1 — Concepts (0–15 min)

Claude for text & content

Claude's greatest strength is structured text generation. Learn how to make it produce exactly the format you need — JSON quizzes, graded answers, and explanations — reliably every time.

🎯

Structured output

Add 'Respond ONLY with valid JSON, no explanation, no markdown' to your prompt. Claude becomes a reliable JSON API.

🌡️

Temperature

0.0 = deterministic, 0.7 = balanced, 1.0 = creative. For quiz generation, use 0.3–0.5. For creative explanations, 0.7.

📝

Chain of thought

Ask Claude to 'think step by step before answering'. For grading and explanations this dramatically improves accuracy.

✅

Output validation

Always validate JSON Claude returns before using it in your UI. Occasionally a token gets dropped. Add a try/catch around JSON.parse().

Part 2 — Hands-on Practice (15–45 min)

Build it

javascript
// Quiz generator — returns clean JSON every time
async function generateQuiz(topic, numQuestions = 5) {
  const prompt = `Create ${numQuestions} multiple-choice quiz questions about: "${topic}"

Respond ONLY with a valid JSON array. No explanation, no markdown, no code blocks.
Format exactly:
[
  {
    "question": "Question text here?",
    "options": ["Option A", "Option B", "Option C", "Option D"],
    "correct": 0,
    "explanation": "Brief explanation of why this answer is correct."
  }
]`;

  const response = await fetch("https://api.anthropic.com/v1/messages", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "x-api-key": process.env.CLAUDE_API_KEY,
      "anthropic-version": "2023-06-01"
    },
    body: JSON.stringify({
      model: "claude-sonnet-4-20250514",
      max_tokens: 2000,
      messages: [{ role: "user", content: prompt }]
    })
  });

  const data = await response.json();
  const text = data.content[0].text.trim();

  // Parse and validate
  const questions = JSON.parse(text); // wrap in try/catch in production
  return questions;
}
Exercise 1Generate and display a quiz

Call generateQuiz('The Water Cycle') and console.log the result. Then write a function that takes this JSON and renders it as HTML buttons in a webpage.

The JSON should have 5 objects each with question, options (array of 4 strings), correct (0-3 index), explanation. To render: loop the array, create a div per question, create a button per option, add a click listener that checks if button index equals correct index.
Exercise 2Grade an answer

Write a separate Claude call that takes a question, the student's chosen answer, and the correct answer, and returns: isCorrect (boolean) and explanation (string) as JSON.

Prompt: 'Question: [Q]. Correct answer: [A]. Student answered: [S]. Respond ONLY with JSON: {"isCorrect": true/false, "explanation": "..."}'. The explanation should be 1-2 sentences explaining the concept whether they got it right or wrong.
Part 3 — Review (45–60 min)

Check your understanding

Answer all questions to see your score.

Q1. What does 'temperature' control in a Claude API call?
✅ Temperature controls creativity. Low = predictable and consistent. High = more varied and creative. For factual quiz generation, keep it low (0.2–0.4).
Q2. You ask Claude for JSON and it returns markdown with ```json code fences. What should your code do?
✅ Strip the fences first: text.replace(/```json|```/g, '').trim() then JSON.parse(). Even better: add 'No markdown. No code fences. Raw JSON only.' to your prompt.
Q3. Why should you validate Claude's JSON output before using it?
✅ Claude is very reliable but not perfect. A single dropped bracket breaks JSON.parse(). Always wrap parsing in try/catch and have a fallback. In production, you might ask Claude to regenerate if parsing fails.
