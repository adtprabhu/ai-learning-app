Part 1 — Concepts (0–15 min)

Testing your AI app

AI apps fail in ways normal software doesn't. Learn how to test an app where outputs are non-deterministic, run 10 real user sessions, and systematically fix what breaks.

🎲

Non-determinism

The same prompt can return different outputs each time. This makes traditional testing ('expect output to equal X') impossible. Test for structure, length, format, and safety — not exact content.

📋

Test categories

(1) Happy path: normal inputs work correctly. (2) Edge cases: empty, too long, unusual characters. (3) Adversarial: attempts to misuse the app. (4) Performance: does it work under load? (5) Regression: does it still work after you change a prompt?

👥

Real user testing

Give 5 real people your app and watch them use it without helping. You will be amazed at what they click, type, and expect that you never anticipated. This is worth 10x more than automated tests.

📝

Bug prioritisation

Rank bugs: Critical (app crashes or gives wrong info), High (key feature broken), Medium (annoying but usable), Low (cosmetic). Fix Critical and High before launch.

Part 2 — Hands-on Practice (15–45 min)

javascript
// Automated test suite for AI app behaviours
// Run with: node tests.js

const { generateQuiz, safeAICall } = require('./your-app');

async function runTests() {
  const results = [];

  async function test(name, fn) {
    try {
      await fn();
      results.push({ name, passed: true });
      console.log(`✅ ${name}`);
    } catch (e) {
      results.push({ name, passed: false, error: e.message });
      console.log(`❌ ${name}: ${e.message}`);
    }
  }

  // Test 1: Quiz generates correct structure
  await test('generateQuiz returns 5 questions', async () => {
    const qs = await generateQuiz('Photosynthesis');
    if (qs.length !== 5) throw new Error(`Got ${qs.length} questions, expected 5`);
    qs.forEach((q, i) => {
      if (!q.question) throw new Error(`Q${i+1} missing question field`);
      if (!Array.isArray(q.options) || q.options.length !== 4) throw new Error(`Q${i+1} wrong options`);
      if (typeof q.correct !== 'number' || q.correct < 0 || q.correct > 3) throw new Error(`Q${i+1} invalid correct index`);
    });
  });

  // Test 2: Empty input handled gracefully
  await test('Empty input returns friendly message', async () => {
    const result = await safeAICall('');
    if (result.success !== false) throw new Error('Should have failed validation');
    if (!result.response.includes('Please')) throw new Error('Response should guide user');
  });

  // Test 3: Very long input handled
  await test('Long input handled gracefully', async () => {
    const longInput = 'a'.repeat(10000);
    const result = await safeAICall(longInput);
    if (result.success !== false) throw new Error('Should have rejected long input');
  });

  // Test 4: Quiz topics produce relevant questions
  await test('Physics quiz contains physics terms', async () => {
    const qs = await generateQuiz('Newton laws of motion');
    const allText = qs.map(q => q.question + q.options.join(' ')).join(' ').toLowerCase();
    const physicsTerms = ['force', 'mass', 'acceleration', 'motion', 'newton', 'velocity'];
    const found = physicsTerms.filter(t => allText.includes(t));
    if (found.length < 2) throw new Error(`Expected physics terms in quiz, found: ${found.join(', ')}`);
  });

  const passed = results.filter(r => r.passed).length;
  console.log(`\n${passed}/${results.length} tests passed`);
  return results;
}

runTests();
Exercise 1Run a 10-person user test

Find 5 students and 5 adults who haven't seen your app. Watch them use it for 10 minutes. Don't explain anything. Just watch and take notes. What confuses them? Where do they click that doesn't work?

Write down every moment of confusion, every wrong click, every feature they expected but wasn't there. Group these into: UI confusion (they didn't know what to do), missing features (they wanted something you didn't build), AI issues (Claude said something wrong or weird). Fix at least 3 things immediately.
Exercise 2Build a prompt regression suite

Every time you change a prompt, run your automated tests to check nothing broke. Create 5 test cases that represent real user scenarios and expected output structure.

Example regression tests: (1) Biology topic generates biology questions (not random topics), (2) Grading returns isCorrect boolean, (3) Coaching insight is encouraging (check for negative words like 'failed', 'terrible', 'awful'), (4) Translation of 'hello' to French returns 'Bonjour', (5) Safety filter catches 'write my essay'.

Q1. Why can't you test an AI app the same way as regular software?
✅ Traditional software: input X always produces output Y. Test that Y equals expected. AI: input X might produce Y, or Y', or Y'' — all valid. Test that the output has the right structure, length, format, and tone — not the exact words.
Q2. You run your app with 5 real users and 4 of them immediately try to click something that doesn't exist. What do you do?
✅ 4/5 users looking for the same thing is very clear product signal. They're not wrong — your app didn't meet their expectation. Either add the feature or dramatically change the UI so they don't expect it. Never fight user expectations.
Q3. What is a 'prompt regression test'?
✅ When you change a prompt to improve one thing, you might accidentally break another. Regression tests catch this: run your test suite after every prompt change to verify existing functionality still works.
