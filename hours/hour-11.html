Part 1 — Concepts (0–15 min)

Chaining AI calls

Complex features need multiple AI steps. Learn to chain calls so the output of one becomes the input of the next — creating multi-step workflows like: generate → test → grade → explain → suggest next topic.

🔗

Why chain?

Some tasks are too complex for one prompt. Breaking them into steps lets each step be optimised. The quiz workflow is: generate questions → receive answers → grade each → explain mistakes → suggest next topic.

⚡

Parallel vs sequential

Some chains can run in parallel (fetch search results AND generate questions at the same time). Others must be sequential (can't grade until student answers). Know which is which.

🛡️

Error handling in chains

If step 2 fails, step 3 should not run. Always validate each step's output before proceeding. Have fallbacks at each stage.

💰

Cost awareness

Each step costs tokens. A 5-step chain might cost 5x a single call. Cache steps where possible. Only chain what's necessary.

// Full quiz chain: generate → display → grade → explain → save → suggest
async function runFullQuizSession(topic, userId) {
  // Step 1: Generate questions (Hour 6)
  const questions = await generateQuiz(topic);

  // Step 2: Display to user (UI handles this, waits for answers)
  const userAnswers = await displayQuizAndWaitForAnswers(questions);

  // Step 3: Grade all answers (can run in parallel!)
  const gradingPromises = questions.map((q, i) =>
    gradeAnswer(q, userAnswers[i])
  );
  const grades = await Promise.all(gradingPromises); // parallel!

  // Step 4: Calculate score
  const score = grades.filter(g => g.isCorrect).length;
  const wrongTopics = grades
    .filter(g => !g.isCorrect)
    .map(g => g.conceptTested);

  // Step 5: Save to Firebase (Hour 10)
  await saveQuizResult(userId, topic, score, questions.length, wrongTopics);

  // Step 6: Get AI suggestion for what to study next
  const suggestion = await suggestNextTopic(topic, wrongTopics);

  return { score, grades, suggestion };
}

// Sequential chain with error handling
async function gradeWithExplanation(question, userAnswer) {
  // Step A: Grade
  let gradeResult;
  try {
    gradeResult = await gradeAnswer(question, userAnswer);
  } catch (err) {
    // Fallback: assume wrong if grading fails
    gradeResult = { isCorrect: false, explanation: "Couldn't grade this answer." };
  }

  // Step B: If wrong, get deeper explanation (only if needed)
  if (!gradeResult.isCorrect) {
    const explanation = await getDeepExplanation(question, gradeResult.explanation);
    return { ...gradeResult, deepExplanation: explanation };
  }

  return gradeResult;
}

async function suggestNextTopic(completedTopic, wrongAreas) {
  const prompt = `A student just completed a quiz on "${completedTopic}".
They struggled with: ${wrongAreas.join(', ') || 'nothing specific'}.
Suggest ONE specific subtopic they should study next. Be brief and encouraging.
JSON: {"suggestion": "...", "reason": "..."}`;

  const response = await fetch("https://api.anthropic.com/v1/messages", {
    method: "POST",
    headers: { "Content-Type": "application/json", "x-api-key": process.env.CLAUDE_API_KEY, "anthropic-version": "2023-06-01" },
    body: JSON.stringify({ model: "claude-sonnet-4-20250514", max_tokens: 150, messages: [{ role: "user", content: prompt }] })
  });
  const data = await response.json();
  return JSON.parse(data.content[0].text.trim());
}

Exercise 2Test parallel vs sequential

Time how long it takes to grade 5 answers sequentially (one after another) vs in parallel with Promise.all(). What is the difference?

Sequential: ~5 × 600ms = 3 seconds. Parallel: ~600ms (all run at the same time). Use Promise.all() whenever steps don't depend on each other. This is one of the biggest easy wins for performance.

Q1. When should you use Promise.all() in an AI chain?
✅ Promise.all() runs multiple async calls simultaneously. Use it when step B doesn't need step A's result to start. Grading 5 answers: all can start at the same time. Generating then grading: sequential, because you can't grade before generating.
Q2. A step in your chain fails mid-way. What should happen?
✅ Each step should have its own try/catch. A failed grading step shouldn't prevent the score being saved. Design chains to be resilient — partial success is better than total failure.
Q3. Why would you NOT put the entire quiz workflow in a single massive prompt?
✅ Giant single prompts are brittle — if anything goes wrong you can't tell where. Chains give you checkpoints: validate after generation, validate after grading. Parallel execution speeds things up. Each step can be unit-tested independently.
