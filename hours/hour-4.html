Part 1 — Concepts (0–15 min)

APIs: the magic bridge between your app and AI

An API (Application Programming Interface) is simply a way for two pieces of software to talk to each other. When your app wants Claude to do something, it doesn't chat via the website — it sends a structured request to the Claude API and gets a structured response back.

Real-world analogy

Imagine a restaurant. You (the app) don't walk into the kitchen and cook yourself. You tell the waiter (the API) what you want. The kitchen (Claude/Google) makes it. The waiter brings it back. You never need to know how the kitchen works — just how to order correctly.

📤

Request

Your app sends a message: 'Claude, here are my instructions, here is the conversation, please respond.' This is an HTTP POST request with your data in JSON format.

📥

Response

Claude sends back: 'Here is my response, here is how many tokens were used, here is why I stopped.' Your app reads this and shows the user the content.

🔑

API Key

A secret password that identifies who is making the request. Anthropic uses this to track usage and charge your account. Never share it publicly.

💰

Pricing

You pay per token — both the tokens you send (input) and the tokens Claude generates (output). Typical cost: fractions of a cent per message. Manageable with smart design.

Anatomy of a Claude API call

javascript
// A complete Claude API call — every field explained
const response = await fetch("https://api.anthropic.com/v1/messages", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    // Your secret API key — NEVER put this in frontend code visible to users
    "x-api-key": "sk-ant-your-key-here",
    "anthropic-version": "2023-06-01"
  },
  body: JSON.stringify({
    model: "claude-sonnet-4-20250514", // which Claude model to use
    max_tokens: 1024,   // maximum length of Claude's response
    system: "You are a helpful study tutor.", // system prompt
    messages: [
      { role: "user", content: "Explain gravity simply." }
    ]
  })
});

// Read the response
const data = await response.json();
const claudeText = data.content[0].text; // the actual reply
const tokensUsed = data.usage.input_tokens + data.usage.output_tokens;
console.log(claudeText);
console.log("Tokens used:", tokensUsed);
⚠️

NEVER put your API key directly in frontend JavaScript that runs in the browser. Anyone can open DevTools and steal it. API keys must live on a server (backend). We will set up proper server-side handling in Hour 16.

HTTP status codes — what can go wrong

✅

200 OK

Success! Claude responded. Read data.content[0].text for the reply.

🔑

401 Unauthorized

Wrong or missing API key. Check your key and headers.

⚡

429 Too Many Requests

You've hit the rate limit. Slow down or add retry logic.

💥

500 Server Error

Something went wrong on Anthropic's side. Retry after a delay.

Exercise 1Your first successful API call

Make a working API call to Claude that asks: 'What are the 3 most important things to know about DNA?' Print the full JSON response. What are the values of: data.model, data.usage.input_tokens, data.usage.output_tokens, data.stop_reason?

data.model: 'claude-sonnet-4-20250514'. input_tokens: ~30 (your prompt). output_tokens: ~150 (Claude's answer). stop_reason: 'end_turn' means Claude naturally finished. 'max_tokens' means it got cut off by your limit. These numbers help you understand and optimise costs.
javascript
// Safer pattern: wrap every API call in try/catch
async function askClaude(question) {
  try {
    const response = await fetch("https://api.anthropic.com/v1/messages", {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
        "x-api-key": process.env.CLAUDE_API_KEY, // from environment variable
        "anthropic-version": "2023-06-01"
      },
      body: JSON.stringify({
        model: "claude-sonnet-4-20250514",
        max_tokens: 1024,
        messages: [{ role: "user", content: question }]
      })
    });

    if (!response.ok) {
      throw new Error(`API error: ${"${response.status}"}`);
    }

    const data = await response.json();
    return data.content[0].text;

  } catch (error) {
    console.error("Claude API failed:", error.message);
    return "Sorry, I couldn't get an answer right now. Please try again.";
  }
}
Part 3 — Review (45–60 min)

Check your understanding

Answer all questions to see your score.

Q1. Where should you store your Claude API key in an app users can access?
✅ Correct! Environment variables on the server. Never in client-side code. Never in URLs. Never in your public GitHub repository. Exposed keys get stolen and abused within hours — it happens constantly to beginners.
Q2. Your API call returns status code 429. What should your code do?
✅ Correct! 429 means 'too many requests'. The right response is exponential backoff — wait 1 second, retry. If 429 again, wait 2 seconds, retry. Then 4 seconds, etc. Most production apps handle this automatically.
Q3. What does max_tokens control?
✅ Correct! max_tokens caps how long Claude's reply can be. Setting it too low cuts off responses mid-sentence. Setting it very high isn't harmful but you pay for every output token, so be reasonable.
