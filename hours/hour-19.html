Part 1 — Concepts (0–15 min)

Deploying to the world

Your app running on localhost is invisible. Google Cloud Run gives it a real URL, auto-scales from 0 to 1000 users, and has a free tier that covers most small apps. Let's ship it.

☁️

Google Cloud Run

Runs your app in a container. Scales automatically — zero instances when no traffic (costs nothing), many instances when busy. Free tier: 2 million requests/month.

🔒

Environment variables in production

Your API keys live in Cloud Run as 'secrets' — not in your code. Set them via the Google Cloud console or CLI. Never commit keys to git.

🐳

Containers (simplified)

A container is a box that includes your app AND everything it needs to run. Cloud Run runs containers. You don't manage servers, OS updates, or infrastructure.

🔗

Custom domains

After deploying, you get a *.run.app URL. You can map your own domain (studyhelper.com) via Google Cloud console. DNS change takes a few minutes.

Part 2 — Hands-on Practice (15–45 min)

Build it

bash
# Step-by-step deployment to Google Cloud Run

# 1. Install Google Cloud CLI (gcloud)
# Download from: cloud.google.com/sdk

# 2. Login and set project
gcloud auth login
gcloud config set project YOUR_PROJECT_ID

# 3. Create a simple server for your app (server.js)
# Your app needs an HTTP server — here's a minimal one:
cat > server.js << 'EOF'
const http = require('http');
const fs = require('fs');
const path = require('path');

const PORT = process.env.PORT || 8080;  // Cloud Run sets PORT automatically

http.createServer((req, res) => {
  // Serve static HTML files
  const filePath = path.join(__dirname, 'public', req.url === '/' ? 'index.html' : req.url);
  const ext = path.extname(filePath);
  const mimeTypes = { '.html': 'text/html', '.js': 'application/javascript', '.css': 'text/css' };

  fs.readFile(filePath, (err, content) => {
    if (err) { res.writeHead(404); res.end('Not found'); return; }
    res.writeHead(200, { 'Content-Type': mimeTypes[ext] || 'text/plain' });
    res.end(content);
  });
}).listen(PORT, () => console.log(`Server running on port ${PORT}`));
EOF

# 4. Create package.json
cat > package.json << 'EOF'
{
  "name": "study-helper",
  "version": "1.0.0",
  "main": "server.js",
  "scripts": { "start": "node server.js" }
}
EOF

# 5. Deploy directly (Cloud Run builds the container for you)
gcloud run deploy study-helper \
  --source . \
  --platform managed \
  --region asia-southeast1 \
  --allow-unauthenticated \
  --set-env-vars CLAUDE_API_KEY=your_key_here,GOOGLE_API_KEY=your_key_here

# 6. Get your live URL
# Cloud Run prints: Service URL: https://study-helper-xxxxx-uc.a.run.app
# Share this URL — your app is live!

# To update after making changes:
gcloud run deploy study-helper --source .
Exercise 1Deploy your app

Follow the steps above. You'll need a Google Cloud account (free tier works). Run the deploy command and wait ~2 minutes. When it's done, you get a real https URL.

Common issues: (1) 'Permission denied' — run gcloud auth login first. (2) 'Port already in use' — Cloud Run auto-sets PORT, make sure your server uses process.env.PORT. (3) App crashes on startup — check Cloud Run logs in the console (Logging tab). Most startup crashes are missing environment variables.
Exercise 2Share and get feedback

Text the URL to 3 people right now. Watch your Google Cloud Run console — you'll see real requests coming in. Ask them: what's confusing, what's missing, what do they love?

This is the moment everything becomes real. Even a rough app in someone else's hands beats a perfect app on localhost. Their immediate reactions (what they click first, what they ask) will tell you more about your product than any amount of solo testing.
Part 3 — Review (45–60 min)

Check your understanding

Answer all questions to see your score.

Q1. What does Google Cloud Run do when your app has zero traffic?
✅ Scale-to-zero is Cloud Run's superpower. When nobody is using your app, you pay nothing. The first request after a period of inactivity takes ~1 second extra (cold start). For most apps this trade-off is perfect — no overnight hosting bills.
Q2. You committed your Claude API key to GitHub by mistake. What must you do immediately?
✅ Revoke immediately — bots scan GitHub for API keys within seconds of a commit. Go to console.anthropic.com → API Keys → Revoke the exposed key → Create a new one → Update your Cloud Run environment variables. Don't just set repo to private — the key is already compromised.
Q3. Your app works perfectly on localhost but crashes on Cloud Run. What should you check first?
✅ Cloud Run logs show exactly what went wrong. Most common causes: missing environment variable (CLAUDE_API_KEY not set in Cloud Run → fix: add it via gcloud run deploy --set-env-vars), wrong PORT (not using process.env.PORT → fix: update server.js), missing npm package (not in package.json → fix: add it and redeploy).
