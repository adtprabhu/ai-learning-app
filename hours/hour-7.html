Part 1 — Concepts (0–15 min)

AI that can see

Modern AI models can read images as naturally as they read text. Learn how to send photos, diagrams, and screenshots to Claude and Gemini, and build a feature that turns any photo of notes into a quiz.

📸

Vision input

Claude accepts images as base64-encoded strings alongside text. The model processes both together — it doesn't 'see' and then 'read', it understands the whole input at once.

📋

What AI can read

Printed text, handwritten notes (if legible), diagrams, charts, equations, screenshots, photos of whiteboards, textbook pages.

⚠️

Limitations

Very small text, poor lighting, extreme handwriting, complex diagrams with tiny labels — accuracy drops. Always let users correct or confirm what the AI extracted.

💡

Use cases

Photo of notes → quiz generator. Photo of equation → step-by-step solution. Screenshot of error → debugging help. Diagram → plain English explanation.

// Send an image to Claude for analysis
async function analyseNotePhoto(imageBase64, mimeType = "image/jpeg") {
  const response = await fetch("https://api.anthropic.com/v1/messages", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "x-api-key": process.env.CLAUDE_API_KEY,
      "anthropic-version": "2023-06-01"
    },
    body: JSON.stringify({
      model: "claude-sonnet-4-20250514",
      max_tokens: 2000,
      messages: [{
        role: "user",
        content: [
          {
            type: "image",
            source: {
              type: "base64",
              media_type: mimeType,   // "image/jpeg", "image/png", "image/webp"
              data: imageBase64       // base64 string, no prefix
            }
          },
          {
            type: "text",
            text: `You are a study tutor. Look at this student's notes.
1. Briefly summarise the main topic (2 sentences).
2. List the 3 most important concepts.
3. Create 3 quiz questions based on the notes.
Return as JSON: { "summary": "...", "concepts": [...], "questions": [...] }`
          }
        ]
      }]
    })
  });
  const data = await response.json();
  return JSON.parse(data.content[0].text.trim());
}

// Convert a File input to base64
function fileToBase64(file) {
  return new Promise((resolve) => {
    const reader = new FileReader();
    reader.onload = () => resolve(reader.result.split(',')[1]);
    reader.readAsDataURL(file);
  });
}
Exercise 1Build the image upload UI

Create an HTML page with a file input (), a preview tag, and a button. When the user picks an image, show the preview. When they click the button, convert to base64 and call analyseNotePhoto().

Use FileReader.readAsDataURL() to get the base64, split on ',' to get just the data part. Display the preview with img.src = reader.result. Show a loading state (disable the button, show 'Analysing...') while the API call runs.
Exercise 2Test with real notes

Take a photo of any page of handwritten or printed notes. Upload it to your app. Does Claude correctly identify the topic? Does it generate sensible quiz questions?

Accuracy is usually high for printed text and good-quality handwritten notes. Common failure cases: very messy handwriting, equations with subscripts, diagrams with tiny labels. For failures, you can add a 'Correct the summary' text field so users can fix errors before generating the quiz.
Part 3 — Review (45–60 min)

Q1. What format does Claude accept images in via the API?
✅ Base64 encoding converts binary image data to a text string that can be sent in JSON. You specify the media type (image/jpeg etc.) so Claude knows how to decode it.
Q2. A student uploads a blurry photo of their notes. Claude's summary is mostly wrong. What is the best fix?
✅ Always give users control over AI output. Show what Claude extracted, let them correct it, then generate the quiz from the corrected text. This pattern (AI suggests, human confirms) is robust and trustworthy.
Q3. What is the practical image size limit you should enforce?
✅ Images cost tokens too — a large high-resolution image uses many more tokens than a smaller one. Compress images client-side before sending (use a canvas resize). Keep uploads under 2–3MB for best performance.
