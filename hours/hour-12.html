Part 1 — Concepts (0–15 min)

Voice with Google

Add a talking tutor to your app. Google's Text-to-Speech reads answers aloud. Google's Speech-to-Text lets students ask questions with their voice. Hands-free studying becomes possible.

🔊

Text-to-Speech (TTS)

Converts Claude's text answers into natural-sounding speech. Google TTS has 200+ voices across 40+ languages. Students can listen to explanations while doing something else.

🎙️

Speech-to-Text (STT)

Records the student's voice and converts it to text. That text is then sent to Claude as the question. The Web Speech API works in-browser without any Google API calls for basic use.

📱

Web Speech API

Built into modern browsers — no API key needed for basic STT. Limited but great for getting started. For production, use Google Cloud Speech-to-Text for better accuracy.

🔑

Google Cloud TTS setup

Enable Cloud Text-to-Speech API in Google Cloud Console. Get an API key. Costs ~$4 per 1 million characters — very cheap for educational use.

// Text-to-Speech — reads Claude's answer aloud
async function speakText(text, languageCode = 'en-US') {
  const response = await fetch(
    `https://texttospeech.googleapis.com/v1/text:synthesize?key=${GOOGLE_TTS_KEY}`,
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        input: { text },
        voice: { languageCode, name: 'en-US-Neural2-F' }, // Neural voices sound best
        audioConfig: { audioEncoding: 'MP3', speakingRate: 0.9 }
      })
    }
  );

  const data = await response.json();
  // data.audioContent is base64-encoded MP3
  const audio = new Audio('data:audio/mp3;base64,' + data.audioContent);
  audio.play();
  return audio; // return so caller can pause/stop it
}

// Speech-to-Text — uses built-in browser Web Speech API (no API key needed!)
function listenForQuestion(onResult, onError) {
  if (!('webkitSpeechRecognition' in window) && !('SpeechRecognition' in window)) {
    onError('Voice input not supported in this browser. Try Chrome.');
    return;
  }

  const SpeechRecognition = window.SpeechRecognition || window.webkitSpeechRecognition;
  const recognition = new SpeechRecognition();

  recognition.lang = 'en-US';
  recognition.interimResults = false; // only final results
  recognition.maxAlternatives = 1;

  recognition.onresult = (event) => {
    const transcript = event.results[0][0].transcript;
    onResult(transcript); // pass the text to your question handler
  };

  recognition.onerror = (event) => onError(event.error);
  recognition.start();

  return recognition; // return so caller can call .stop() if needed
}

// Combined: listen → ask Claude → speak answer
async function voiceStudySession() {
  const micBtn = document.getElementById('mic-btn');
  micBtn.textContent = '🎙️ Listening...';
  micBtn.disabled = true;

  listenForQuestion(
    async (question) => {
      micBtn.textContent = '💭 Thinking...';
      const answer = await askClaude(question); // your Hour 4 function
      document.getElementById('answer-display').textContent = answer;
      micBtn.textContent = '🔊 Speaking...';
      const audio = await speakText(answer);
      audio.onended = () => { micBtn.textContent = '🎙️ Ask a question'; micBtn.disabled = false; };
    },
    (error) => { micBtn.textContent = '🎙️ Ask a question'; micBtn.disabled = false; alert('Voice error: ' + error); }
  );
}

Q1. The Web Speech API for voice input requires:
✅ The Web Speech API is built into Chrome, Edge, and Safari. No API key, no library, no cost. It's not perfect (limited language support, no custom vocabulary) but it's ideal for getting started quickly.
Q2. Google Cloud TTS pricing is ~$4 per 1 million characters. A student session uses roughly 2,000 characters of spoken text. What does voice cost per session?
✅ 2,000 / 1,000,000 × $4 = $0.008. Less than a cent per session. Voice is one of the cheapest Google APIs to run. Even at 1,000 students per day, voice costs are under $10/day.
Q3. A student clicks the microphone but nothing happens. What should you check first?
✅ Browser microphone permission is the #1 cause of voice input failures. Check: navigator.permissions.query({name:'microphone'}) and guide users to allow it if denied. Always have a fallback text input for users who can't use voice.
