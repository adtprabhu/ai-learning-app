Part 1 — Concepts (0–15 min)

Speed & cost optimisation

A slow expensive app dies. Learn to stream Claude's responses for instant feel, cache repeated questions to cut costs by 90%, and track your API spend before it surprises you.

🌊

Streaming

Instead of waiting for Claude's full response (2–4 seconds), stream it word by word. The user sees text appearing immediately. Feels 10x faster even though total time is similar.

💾

Caching

If 100 students ask about photosynthesis, call the API once and serve the cached result to the other 99. Use a simple object as a cache, or Firebase for persistence.

📊

Token budgets

Set max_tokens deliberately. A quiz question needs ~400 tokens. A full explanation needs ~600. Don't set it to 4096 for everything — unused budget still adds latency.

🔍

Monitor costs

Check the Anthropic console usage dashboard weekly. Set budget alerts. Know your cost per session. This is basic product management.

// 1. STREAMING — show response word by word
async function streamAnswer(question, onChunk, onComplete) {
  const response = await fetch("https://api.anthropic.com/v1/messages", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "x-api-key": process.env.CLAUDE_API_KEY,
      "anthropic-version": "2023-06-01"
    },
    body: JSON.stringify({
      model: "claude-sonnet-4-20250514",
      max_tokens: 600,
      stream: true,  // ← enable streaming
      messages: [{ role: "user", content: question }]
    })
  });

  const reader = response.body.getReader();
  const decoder = new TextDecoder();
  let fullText = '';

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    const chunk = decoder.decode(value);
    const lines = chunk.split('\n').filter(l => l.startsWith('data: '));

    for (const line of lines) {
      const data = JSON.parse(line.slice(6));
      if (data.type === 'content_block_delta' && data.delta?.text) {
        fullText += data.delta.text;
        onChunk(data.delta.text);  // call UI update with each new word
      }
      if (data.type === 'message_stop') onComplete(fullText);
    }
  }
}

Q1. What is the main user experience benefit of streaming?
✅ Streaming doesn't speed up Claude — it changes when you see the output. Instead of waiting 3 seconds to see anything, you see text start in 200ms. Psychologically, this feels dramatically faster. Netflix uses this same principle with progressive video loading.
Q2. You cache quiz questions for 'The Water Cycle'. 500 students use the app today, 80% asking about this topic. How many API calls do you make?
✅ One real API call, then 399 served from cache. 99.75% cost reduction for that topic. This is why caching is the single highest-impact optimisation for educational apps with predictable, repeated queries.
Q3. What should you do when your monthly API cost doubles unexpectedly?
✅ Investigate before acting. The dashboard shows per-model, per-day usage. Find the expensive calls — often a single prompt type accounts for 60%+ of cost. Optimise max_tokens for that prompt type, add caching, or redesign that feature. Targeted fixes beat blunt cuts.
