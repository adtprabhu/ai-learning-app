Part 1 — Concepts (0–15 min)

Google Search + AI

AI knowledge has a cutoff date — it doesn't know about recent events. Google Search gives it live information. Learn to connect both so your app gives accurate, current answers on any topic.

📅

Knowledge cutoff

AI models were trained on data up to a specific date. Ask about recent events and they'll either say 'I don't know' or confidently give outdated information.

🔍

Grounding

The technique of giving AI access to real, retrieved information before asking it to answer. Dramatically improves factual accuracy.

🔗

Google Custom Search API

Free tier: 100 searches/day. Returns structured JSON with titles, URLs, and snippets. Easy to set up via Google Cloud Console.

🧠

The RAG pattern preview

Search → retrieve relevant text → inject into Claude's context → ask Claude to answer based on retrieved text. This is the foundation of Hour 9.

Part 2 — Hands-on Practice (15–45 min)

Build it

javascript
// Step 1: Search Google for relevant content
async function searchGoogle(query) {
  const API_KEY = process.env.GOOGLE_SEARCH_API_KEY;
  const SEARCH_ENGINE_ID = process.env.GOOGLE_SEARCH_ENGINE_ID;

  const url = `https://www.googleapis.com/customsearch/v1?key=${API_KEY}&cx=${SEARCH_ENGINE_ID}&q=${encodeURIComponent(query)}&num=3`;

  const response = await fetch(url);
  const data = await response.json();

  // Extract just the useful parts
  return data.items.map(item => ({
    title: item.title,
    snippet: item.snippet,
    url: item.link
  }));
}

// Step 2: Ask Claude to answer using the search results
async function searchAndAnswer(userQuestion) {
  // First, get real-world information
  const results = await searchGoogle(userQuestion);

  // Format the results for Claude
  const context = results.map((r, i) =>
    `Source ${i+1}: ${r.title}\n${r.snippet}`
  ).join('\n\n');

  // Now ask Claude, giving it the real information
  const prompt = `Answer this student's question using ONLY the provided sources.
If the sources don't contain the answer, say so honestly.

SOURCES:
${context}

STUDENT QUESTION: ${userQuestion}

Give a clear, age-appropriate answer in 2-3 sentences. Cite which source you used.`;

  const response = await fetch("https://api.anthropic.com/v1/messages", {
    method: "POST",
    headers: { "Content-Type": "application/json", "x-api-key": process.env.CLAUDE_API_KEY, "anthropic-version": "2023-06-01" },
    body: JSON.stringify({ model: "claude-sonnet-4-20250514", max_tokens: 500, messages: [{ role: "user", content: prompt }] })
  });

  const data = await response.json();
  return { answer: data.content[0].text, sources: results };
}
Exercise 1Set up Google Custom Search

Go to console.cloud.google.com, enable the Custom Search API, get an API key. Then go to programmablesearchengine.google.com and create a search engine. Note your Search Engine ID. Test it by calling the search URL directly in your browser.

You should see JSON with an 'items' array. Each item has title, link, snippet. The snippet is the short preview text — this is what we feed to Claude as context.
Exercise 2Compare grounded vs ungrounded

Ask Claude directly: 'What is the most recently discovered exoplanet?' Then use your searchAndAnswer() function with the same question. Compare accuracy and specificity.

Without search: Claude gives a vague answer from its training data, possibly outdated. With search: Claude cites a specific planet from a recent news article. This difference is why grounding matters for any time-sensitive topic.
Part 3 — Review (45–60 min)

Check your understanding

Answer all questions to see your score.

Q1. What problem does 'grounding' AI with search solve?
✅ Grounding solves the knowledge cutoff problem. By retrieving fresh information and injecting it into the prompt, you can ask Claude about events from yesterday — it will answer accurately from the retrieved text.
Q2. In your searchAndAnswer() function, why do we tell Claude to 'answer using ONLY the provided sources'?
✅ Without this instruction, Claude might blend training knowledge with retrieved knowledge, potentially still using outdated information. 'Only use sources' forces it to stay grounded.
Q3. Your Google Custom Search free tier allows 100 searches per day. You have 200 students. What should you add?
✅ Caching is the right answer. If 50 students ask about 'photosynthesis' in one day, search Google once, save the result, serve the cached result to the other 49. This dramatically reduces API calls. We implement caching in Hour 17.
